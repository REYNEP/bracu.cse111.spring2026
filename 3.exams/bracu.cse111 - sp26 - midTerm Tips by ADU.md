#### Some suggestions for the midterm exam,
- Be at the exam hall at least 30 minutes before the exam. Do not be late.
- Bring your own calculator, pen, pencil, eraser, ruler and other materials you deem necessary for the exam. 
- Very carefully read the questions first. Try to figure out a rough solution and then go for the answer.
- For reverse engineering questions,
    - See how many instance variables are needed by going through the expected output.
    - Then check how many methods are being called from the tester code and check which methods require overloading. Try to figure out the differences in those methods. Remember, one method being called multiple times doesn't necessarily mean the method is overloaded.
    - To find overloading, look for methods with the same name but different parameter count/different datatype/different sequence of datatypes for parameters.
    - Identify whether a method is a void or return type.
    - Check the number of created objects to see whether constructor overloading is required or not. 
    - The main difference between a variable and a method is parenthesis. obj.method is not a method and obj.var() is not a variable.
    - Follow the given dummy lines in tester code to figure out the functionality of a method.
    - For multi-class questions, find out which methods belong to which class first and then design the classes.
    - Read the description to find out whether you need private variables/methods. No need to use private access modifier if it is not mentioned in the question. 

- For tracing,
    - Properly draw the table.
    - Assign address to every object and array.
    - Tables should be designed properly so that you can distinguish between instance variables, instance methods and local variables.
    - Finally write the output in the question paper.

- Get a good sleep before the exam. A fresh mind helps a lot more than you think.

Wishing you all the best!
